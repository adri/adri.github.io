Deployments

Deployments vs Pods? Separate things. Does a deployment overwrites Pods? It kills it and creates a new one Deploy with

  • kubectl set image deployment/some-deployment some-image=vendor/some-image:tag. e.g. kubectl set image deployment/todo todo-php=ticketswap/todo-php:latest
  • kubectl rollout status deployment [deployment]
  • kubectl rollout history deployment [deployment], set revisionHistoryLimit: 100
  • kubectl rollout undo [deployment]

Ingresses

Differences to AWS EC

  • Pod vs Task Almost the same. You also have port definition, containers and health. Problem with AWS tasks with terraform: New versions of tasks are created when deploying. Terraform doesn't know about them and wants to roll back when terraform files change. What could be different with K8s: Deployments?, Services without Pods?
  • More management compared to AWS (more software to take care of)
  • Kubelet: Similar agent concept to AWS
  • AWS has many log options but no easy log access interface
  • Cron Jobs from Kubernetes seems nice, not sure if AWS has it
