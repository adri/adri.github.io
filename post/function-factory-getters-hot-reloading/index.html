NodeJS: Factory Function Pattern, Dependency Registry and Hot Reloading

2016-01-20 — 1 min read

Factory Functions Pattern are a great way to structure applications in NodeJS. This In-Depth article describes how they are used. In short, the advantages of this pattern are:

  • All external dependencies of a module are stated explicitly
  • External dependencies can be replaced without changing the module
  • Unittesting works by passing in mocked/testing dependencies
  • No global references
  • Inversion of control

This post shows how to:

  1. Code using the Factory Function pattern
  2. Wire up dependencies in one place
  3. Hot reload a server for fast iteration

Wiring up dependencies

More complex NodeJS applications can gain maintainability and testability from such a structure.
Dependencies of the app can be wired up in one place.

An example which runs with NodeJS > 4.x looks like this:

1// In file ./index.js
2
3// Wire up dependencies
4const leveldb = require("leveldb");
5const express = require("express");
6const User = require("./model/User")();
7const userRepository = require("./repository/UserRepository")(leveldb);
8const userService = require("./service/UserService")(userRepository, User);
9const userController = require("./controller/UserController")(userService);
10
11const app = express();
12// Attach routes
13userController.routes(app);
14
15app.listen(3000, () => console.log("Listening on http://localhost:3000"));

Example UserService implementation

To illustrate how injected dependencies can be used, an example implementation of a UserService could be something like this:

1// In file ./src/service/UserService.js
2
3// userRepository and User are external dependencies.
4module.exports = (userRepository, User) => {
5  return {
6    /**
7     * Login a user with the given login provider and user profile.
8     *
9     * @param  {string} provider Login provider which was used (google, facebook, ...)
10     * @param  {Object} profile User profile from that provider.
11     * @return {User} Logged in user.
12     */
13    loginWithProvider: (provider, profile) => {
14      return (
15        userRepository
16          .findByProvider(provider, profile)
17          // If not found, this is a new user.
18          .then((user) => user || User.fromProvider(provider, profile))
19          .then((user) => user.updateLastLogin())
20          .then((user) => userRepository.store(user))
21      );
22    },
23  };
24};

There are two injected dependencies, the userRepository and a User model. These dependencies can be accessed in the returned object with currently one function loginWithProvider. Other function could be added here, they would share the same dependencies.

Example usage of the UserService

A wired up instance of the example UserService can then be used in other places of the app, like for example
controllers. The same principles applies there:

1// In file ./src/controllers/UserController.js
2
3module.exports = (userService) => {
4  function login(req, res, next) {
5    return userService
6      .loginWithProvider(req.query.provider, req.query.profile)
7      .then((user) => res.json(user))
8      .catch(next);
9  }
10
11  return {
12    routes: (app) => {
13      app.get("/login", login);
14    },
15  };
16};

Now all dependencies are injected and wired up at one place. Nice!

Refactor to support Hot Reloading

When apps grow, the startup cost increases due to parsing javascript files
and making connections to databases and servies. Hot reloading NodeJS apps avoids the startup costs and
lets you iterate quicker. To make this possible the previous structure needs a little refactoring.

We have to move out the dependencies into another file which can be reloaded on file changes.
The dependencies then need to be exported to be used:

1// In ./src/dependencies.js
2
3// Static dependencies, changing these requires a restart
4// in hot reloading mode.
5const level = require("level");
6const leveldb = require("level-promisify")(level);
7const express = require("express");
8
9const app = express();
10
11const deps = {
12  // Dynamic dependencies which will be hot reloaded.
13  // Use ES6 getters to require when needed.
14
15  app,
16
17  get routes() {
18    deps.userController.routes(deps.app);
19    return app;
20  },
21
22  get User() {
23    return require("./model/User")();
24  },
25  get userRepository() {
26    return require("./repository/UserRepository")(leveldb);
27  },
28  get userService() {
29    return require("./service/UserService")(deps.userRepository, deps.User);
30  },
31  get userController() {
32    return require("./controller/UserController")(deps.userService);
33  },
34};
35
36module.exports = deps;

This is how the ./index.js looks like after the refactoring:

1// In file ./index.js
2
3// Start with hot reloading: node index hotreload
4const isHot = process.argv[2] === "hotreload";
5const app = isHot ? require("./hotreload") : require("./src/server");
6
7app.listen(3000, () => console.log("Listening on http://localhost:3000"));
1// In file ./src/server.js
2
3const dependencies = require("./dependencies");
4const app = dependencies.app;
5const routes = dependencies.routes;
6
7app.use(routes);
8
9module.exports = app;

For hot reloading, create a new file:

1// In file ./src/hotreload.js
2const chokidar = require("chokidar");
3const dependencies = require("./src/dependencies");
4const app = dependencies.app;
5
6// Include server routes as a middleware
7app.use((req, res, next) => {
8  // Require all dependencies.
9  dependencies.routes(req, res, next);
10});
11
12// Do "hot-reloading" of express stuff on the server
13// Throw away cached modules and re-require next time
14// Ensure there's no important state in there!
15const watcher = chokidar.watch(["./src"]);
16watcher.on("ready", () => {
17  watcher.on("all", () => {
18    console.log("Clearing module cache from server");
19    Object.keys(require.cache).forEach((file) => {
20      if (/src/.test(file) && file.indexOf("node_modules") === -1) {
21        delete require.cache[file];
22      }
23    });
24  });
25});
26
27module.exports = app;
